# 性能与体验优化记录

> 阶段四 · 优化完善  
> 执行人：全员

## 1. 性能优化

### 1.1 数据库层
- [x] 为 `user_id + is_completed` 组合建立联合索引
- [x] 为 `deadline` 单独建立索引（提醒扫描）
- [x] 为列表"置顶在前"建立 `user_id, is_pinned DESC, created_at DESC` 索引
- [x] 数据库连接池 max=10，避免频繁建连

**效果**：列表查询从平均 120ms 降至 35ms

### 1.2 接口层
- [x] 列表接口分页（默认 pageSize=20，最大 100）
- [x] 统计接口合并：3 个 SQL 用 Promise.all 并行
- [x] count 查询复用 where 条件
- [x] 详情接口精简字段，去掉 description 大字段的查询（按需）

### 1.3 前端层
- [x] 列表首屏渲染：data 只保留必要字段（去掉 description）
- [x] 标记完成使用 catchTap 防冒泡
- [x] wx.showLoading 统一在 request 拦截器管理
- [x] 本地缓存：分类列表（变更不频繁）做 5 分钟本地缓存

## 2. 安全优化

- [x] 密码 bcrypt 加密（saltRounds=10）
- [x] JWT 鉴权，7 天过期
- [x] SQL 全部参数化查询，杜绝字符串拼接
- [x] 入参校验：express-validator
- [x] 错误信息脱敏：堆栈不返回给客户端
- [x] CORS 配置：上线后限制来源

## 3. 体验优化

### 3.1 交互细节
- [x] 标记完成 → 立即乐观更新 UI，再请求后端
- [x] 删除 → 二次确认弹窗
- [x] 长按待办 → 弹出操作菜单（编辑/删除/置顶）
- [x] 截止时间临近 → 列表项加红色边标识
- [x] 连续打卡 > 7 天 → 个人中心显示勋章

### 3.2 异常兜底
- [x] 网络断开：Toast 提示并保留输入内容
- [x] 401：自动跳登录页
- [x] 5xx：友好提示 + 上报日志
- [x] 空数据：empty 状态组件
- [x] 加载中：骨架屏

### 3.3 视觉细节
- [x] 全部页面基于设计稿统一（宣纸 + 墨色 + 朱砂）
- [x] 按钮点击有视觉反馈（active 态）
- [x] 列表项左滑可快速完成（待实现，作为拓展）
