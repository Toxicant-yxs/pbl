# 调研笔记 · 个人待办事项管理小程序

> 阶段一：选题分工与资料搜集（1 课时）  
> 整理人：王五  
> 审核人：张桂莲

## 1. 业务背景

随着高校学生学习、生活节奏加快，碎片化任务越来越多。调研显示：
- **82%** 的大学生表示"经常忘记要做的事"（N=120，校内问卷）
- 主流待办 App（滴答清单、Todoist、Microsoft To Do）功能丰富但学习成本高、广告多
- 微信小程序"即开即用、无需下载"的特点天然契合"快速记录"场景

## 2. 竞品分析

| 产品 | 优点 | 缺点 | 借鉴点 |
|------|------|------|--------|
| 滴答清单 | 跨平台、清单层级清晰 | 高级功能付费、广告多 | 任务层级、标签系统 |
| Microsoft To Do | 界面简洁、我的日视图 | 国内访问慢、无提醒推送弱 | "我的一天"视图设计 |
| 锤子便签 / 极简待办 | 极简、专注 | 无统计、无提醒 | 极简输入交互 |
| 微信收藏 + 提醒 | 零成本 | 无结构化、无统计 | 集成到微信生态 |

**结论**：做一款"轻量、聚焦、统计可视化"的微信小程序待办，专注学生群体。

## 3. 技术预研

### 3.1 微信小程序
- 基础库 ≥ 2.19.0，支持 wx.request 拦截器、组件 slot
- 订阅消息：用户授权后可在截止前推送，单次授权只能发 1 条 → 需要在每次新建待办时引导授权
- 本地存储：wx.setStorageSync 存储 token 与用户信息

### 3.2 Node.js + Express
- Node.js v18 LTS，Express 4.x
- 中间件：cors、jsonwebtoken、bcrypt、express-validator、morgan、express-async-errors
- 定时任务：node-cron 每分钟扫描待提醒记录

### 3.3 MySQL 8.0
- 4 张核心表：users、categories、todos、reminders
- 索引：user_id、is_completed、deadline
- 连接池：mysql2/promise，max=10
