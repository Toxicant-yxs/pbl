## 1. 引言

### 1.1 项目背景
随着高校学生学习节奏加快，碎片化任务越来越多。调研显示 82% 的大学生存在"经常忘记要做的事"的困扰。现有待办类应用（滴答清单、Todoist、Microsoft To Do）功能丰富但学习成本高、广告多，与学生群体"轻量、专注"的需求存在错位。

微信小程序"即开即用、无需下载"的特点，天然契合"快速记录"场景。本项目基于 PBL 训练营第 20 号课题，开发一款轻量、聚焦、统计可视化的微信小程序待办工具。

### 1.2 项目目标
- 实现《PBL 训练营实训计划》九.20 课题的全部 7 项基础功能
- 覆盖"用户管理—待办增删改查—统计—提醒—个人中心"完整业务闭环
- 形成可演示、可答辩、可作为团队作品集的项目

### 1.3 技术选型
| 层 | 技术 | 选型理由 |
|----|------|----------|
| 客户端 | 微信小程序原生 | 实训要求、零学习成本、跨端 |
| 服务端 | Node.js 18 + Express 4 | 实训要求、生态成熟、上手快 |
| 数据库 | MySQL 8.0 | 实训要求、事务完善、运维简单 |
| 鉴权 | JWT | 无状态、适合移动端 |
| 加密 | bcrypt | 业界标准 |
| 定时 | node-cron | 轻量、API 友好 |
| 部署 | 本地 + 可迁移云 | 训练营环境 |

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 用户模块
- 用户注册：用户名（4-20 字符，唯一）、密码（6-20 字符）、昵称
- 用户登录：账号密码登录，返回 JWT
- 个人信息：查看、编辑

#### 2.1.2 待办模块（核心）
- **增**：标题（必填，1-200 字符）、描述、分类、优先级、截止时间、提醒时间、置顶
- **删**：单条删除，二次确认
- **改**：编辑全部字段、标记完成、切换置顶
- **查**：
  - 列表（默认按"置顶 > 创建时间"倒序）
  - 状态筛选：全部 / 待完成 / 已完成
  - 分类筛选
  - 优先级筛选
  - 时间范围筛选：今天 / 明天 / 本周 / 已过期
  - 关键词搜索（title + description 模糊匹配）

#### 2.1.3 分类模块
- 系统提供 4 个默认分类：学习 / 工作 / 生活 / 健康
- 用户可自定义增删改
- 删除分类时，其下待办的 category_id 自动置为 NULL

#### 2.1.4 统计模块
- 总览：总数、已完成、未完成、完成率、连续打卡天数
- 趋势：近 7/30 日每日完成数量
- 分类分布：各分类的待办数量
- 连续打卡：从今天向前连续每天至少完成 1 个

#### 2.1.5 提醒模块
- 用户新建带提醒的待办时，引导订阅消息授权
- 后端 node-cron 每分钟扫描 `remind_at <= NOW() AND is_sent = 0` 的记录
- 推送到用户微信

#### 2.1.6 个人中心
- 头像、昵称、UID
- 总待办、已完成、总完成率
- 入口：我的待办、统计、分类、提醒设置、账号、退出

### 2.2 非功能需求

| 维度 | 指标 |
|------|------|
| 性能 | 核心接口 P95 ≤ 500ms，列表首屏 ≤ 1.5s |
| 安全 | 密码 bcrypt 加密、JWT 鉴权、参数化 SQL |
| 兼容 | 微信基础库 ≥ 2.19.0 |
| 易用 | 关键操作 3 步内可达 |
| 维护 | MVC 分层、接口文档化、错误码统一 |


### 3.4 关键流程

#### 3.4.1 标记完成流程
```
用户点击圆圈
   ↓
catchTap 防穿透
   ↓
乐观更新 UI（立即划线）
   ↓
PATCH /api/todos/:id/complete
   ↓
后端 UPDATE is_completed=1, completed_at=NOW()
   ↓
返回 200
   ↓
重新拉取列表（同步统计）
```

#### 3.4.2 提醒推送流程
```
用户新建带提醒的待办
   ↓
引导 wx.requestSubscribeMessage
   ↓
写入 reminders (is_sent=0)
   ↓
[node-cron 每分钟扫描]
   ↓
匹配 remind_at <= NOW() AND is_sent=0
   ↓
调用微信订阅消息 API
   ↓
UPDATE reminders SET is_sent=1, sent_at=NOW()
```

#### 4.1.2 关键技术
- **request 封装**：统一 baseURL、Token 注入、401 跳转、错误提示

#### 4.1.3 关键代码（节选）
```javascript
// utils/request.js
function request({ url, method = 'GET', data, auth = true }) {
  wx.showLoading({ title: '加载中', mask: true });
  const header = { 'Content-Type': 'application/json' };
  if (auth) {
    const token = wx.getStorageSync('token');
    if (token) header.Authorization = 'Bearer ' + token;
  }
  return new Promise((resolve, reject) => {
    wx.request({
      url: app.globalData.apiBase + url,
      method, data, header,
      success: (res) => {
        wx.hideLoading();
        if (res.statusCode === 401) {
          wx.removeStorageSync('token');
          wx.reLaunch({ url: '/pages/login/login' });
          return reject(res.data);
        }
        if (res.data.code === 0) return resolve(res.data.data);
        wx.showToast({ title: res.data.msg });
        reject(res.data);
      },
      fail: (err) => {
        wx.hideLoading();
        wx.showToast({ title: '网络异常' });
        reject(err);
      }
    });
  });
}
```

### 4.2 后端实现（Node.js + Express）

#### 4.2.1 项目结构
```
server/
├── app.js (入口)
```

#### 4.2.2 关键技术
- **连接池**：mysql2/promise，max=10
- **统一错误处理**：app.js 全局错误中间件
- **参数化查询**：杜绝 SQL 注入
- **JWT 中间件**：routes 前置 auth
- **定时任务**：node-cron 每分钟扫描

#### 4.2.3 关键代码（节选）
```javascript
// middleware/auth.js
module.exports = (req, res, next) => {
  const token = (req.headers.authorization || '').replace(/^Bearer\s+/, '');
  if (!token) return res.status(401).json({ code: 401, msg: '未提供 Token' });
  try {
    const payload = jwt.verify(token, secret);
    req.user = { id: payload.id, username: payload.username };
    next();
  } catch (e) {
    res.status(401).json({ code: 401, msg: 'Token 无效或已过期' });
  }
};

// utils/scheduler.js
cron.schedule('* * * * *', async () => {
  const [rows] = await pool.query(
    `SELECT r.*, t.title FROM reminders r
     JOIN todos t ON r.todo_id = t.id
     WHERE r.is_sent = 0 AND r.remind_time <= NOW() AND t.is_completed = 0`
  );
  for (const r of rows) {
    await sendWechatSubscribe(r); // 实际调用微信 API
    await pool.query('UPDATE reminders SET is_sent=1, sent_at=NOW() WHERE id=?', [r.id]);
  }
});
```

### 4.3 数据库实现

#### 4.3.1 建表
```sql
CREATE TABLE todos (
  id INT NOT NULL AUTO_INCREMENT,
  user_id INT NOT NULL,
  category_id INT DEFAULT NULL,
  title VARCHAR(200) NOT NULL,
  description TEXT,
  priority TINYINT NOT NULL DEFAULT 0,
  deadline DATETIME DEFAULT NULL,
  remind_at DATETIME DEFAULT NULL,
  is_completed TINYINT NOT NULL DEFAULT 0,
  is_pinned TINYINT NOT NULL DEFAULT 0,
  completed_at DATETIME DEFAULT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  KEY idx_user_completed (user_id, is_completed),
  KEY idx_pinned (user_id, is_pinned DESC, created_at DESC),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 6. 总结与展望

### 6.1 项目总结
本项目基于"项目驱动、实操落地"理念，完整实现个人待办事项管理小程序的全部 7 项基础功能。技术栈覆盖微信小程序前端、Node.js+Express 后端、MySQL 数据库、JWT 鉴权、定时任务等企业级开发常见技术。

通过 8 课时的训练营实战，团队完整走通"选题—调研—需求—设计—开发—测试—优化—答辩"的全流程，积累了宝贵的项目经验。

### 6.2 主要收获
- **技术层面**：掌握了微信小程序原生开发、Express 路由分层、MySQL 索引优化、JWT 鉴权
- **工程层面**：学会了 Git 分支协作、接口文档先行、测试用例设计、性能压测
- **协作层面**：体验了 4 人小团队分工、每日站会、问题追踪与解决

### 6.3 不足与改进方向

| 现状 | 改进方向 |
|------|----------|
| 无单元测试 | 引入 Jest，覆盖 service/dao 层 |
| 无 CI/CD | 接入 GitHub Actions 自动化部署 |
| 无日志系统 | 接入 Winston + ELK |
| 无监控告警 | 接入 Sentry 异常监控 |
| 单机数据库 | 接入主从复制 / Redis 缓存 |

### 6.4 功能拓展
- 番茄钟专注模式
- 多人协作待办（邀请好友）
- AI 智能推荐完成时间

- 暗色模式
- 微信支付 / 会员订阅
