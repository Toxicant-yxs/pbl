## 一、需求分析

### 1.1 功能需求（来自实训计划九.20 基础功能）

| 编号 | 功能模块 | 说明 |
|------|----------|------|
| F1 | 用户注册/登录 | 账号密码登录、注册、Token 鉴权 |
| F2 | 待办添加 | 内容、优先级（紧急/中等/较低/无）、截止时间、分类、备注 |
| F3 | 待办管理 | 标记完成、编辑、删除、置顶 |
| F4 | 待办查询 | 按分类/优先级/截止时间筛选、关键词搜索 |
| F5 | 待办统计 | 已完成/未完成数量、完成率、连续打卡、趋势 |
| F6 | 提醒功能 | 截止时间前消息提醒（订阅消息/本地通知） |
| F7 | 个人中心 | 待办记录、统计、个人信息、设置 |

### 1.2 非功能需求

- **性能**：核心接口响应 ≤ 3 秒，列表首屏加载 ≤ 1.5 秒
- **安全**：密码 bcrypt 加密、参数校验、SQL 注入防护、JWT 鉴权
- **兼容**：微信小程序基础库 ≥ 2.19.0
- **异常处理**：网络异常、输入非法、接口失败、数据库异常均有友好提示

## 二、系统设计

### 2.1 架构

采用前后端分离三层架构：

```
小程序端 (WeChat Mini Program)
    ↓ wx.request / HTTPS / JSON
Node.js + Express 后端 (RESTful API)
    ↓ mysql2 / mongoose
MySQL / MongoDB 数据库
```

### 2.3 接口设计（RESTful API 全部以 `/api` 为前缀）

## 四、核心实现要点

### 4.1 异常处理
- 全局 try/catch + 统一错误中间件
- 前端 wx.request 封装统一拦截 401/500
- 数据库连接异常自动重试（最多 3 次）

### 4.2 安全
- 密码：bcrypt（saltRounds=10）
- 鉴权：JWT，Token 有效期 7 天
- 参数：express-validator 校验
- SQL：使用参数化查询，杜绝字符串拼接
- 敏感字段：登录密码不返回

### 4.3 性能
- 数据库连接池（max=10）
- 列表分页（默认 pageSize=20）
- 高频查询字段加索引
- 小程序端首屏 SSR + 骨架屏

### 4.4 提醒功能
- 后端 node-cron 每分钟扫描 `remind_at <= NOW() AND is_sent = 0`
- 调用微信订阅消息 API 推送
- 用户首次添加待办时引导订阅消息授权

### 5.2 典型问题与解决（建议准备 3-5 个）

1. **小程序请求后端跨域**：开发期勾选"不校验合法域名"；上线后在后端配置 cors 白名单仅允许 `servicewechat.com`。
2. **Token 过期处理**：在 request.js 拦截 401，跳转登录页并清除本地存储。
3. **订阅消息一次性**：用户每次拒绝后下次需重新引导授权，使用 `wx.requestSubscribeMessage`。
4. **时间格式**：数据库存 DATETIME，API 统一返回 ISO 8601（+08:00），前端按本地时区展示。
5. **列表性能**：分页 + 下拉刷新 + 字段精简，避免一次返回 description 大字段。
